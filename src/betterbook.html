BetterBook

BetterBook is a full-stack application aimed at providing a platform for single-theatre operations to manage online ticketing, promotions, and showtimes. This application was built alongside a partner using Spring Boot, MySQL, and React to effectively store and retrieve data.

Process

This project was developed using an Agile approach. Our development included 3 sprints where we iteratively improved each area of our application.

PHASE 1 - PLANNING

First, we had to establish user stories to ensure we met the needs of all customers and vendors. Then, we built a backend API using the Spring Boot Web JPA to connect our frontend to our database. We also defined API endpoints, database schema, and authentication mechanisms.

PHASE 2 - DEVELOPMENT

During this phase, we implemented the core features and integrated the frontend, backend, and database.

  • Built an admin panel to manage content, like a mini home-built CMS (Content Management System).
  • Implemented RESTful APIs for user log-in, sign-up, password encryption, and password reset.
  • Added email verification, promotions, and payment processing.
  • Designed the MySQL database schema to store movie data, user information, bookings, and promotions.

PHASE 3 - OPTIMIZATION

Before launching, we performed thorough testing and optimizations to ensure a smooth user experience. These improvements were made while keeping our initial user stories in mind. We constructed a plan to deploy BetterBook using Netlify.

Problems, thought process, and what I learned.

Like most projects, I ran into a few bumps along the way. A few difficult areas were maintaining project structure, dependency management, and API security.

Due to the project's sheer size and multiple hands working on it, it was essential to construct reusable components and hooks so we could iterate quickly and clearly. If I had to change anything, I would have opted for a simplified database using something like Flask, MongoDB, or Laravel paired with Clerk to handle authentication and security, and data in one fell swoop.

Through this project, I learned a lot about what goes into planning, building an application that fulfills the needs of users, admins, and clients. I think this knowledge will be applicable in working through real-world case studies with clients.
